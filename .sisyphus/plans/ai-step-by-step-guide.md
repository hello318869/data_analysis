# AI 渐进式协作指南 — 交互式数据分析系统

## TL;DR

> **一句话概述**：把整个数据分析系统拆成 20 个最小步骤，每步让 AI 只写 ≤50 行代码并逐行解释。你理解一步、验证一步、再走下一步——最终系统建成的同时你也真学会了。
>
> **核心纪律**：
> - 每次只问一个功能点，绝不让 AI 一口气吐几百行
> - 拿到代码先让 AI 解释，再自己跑，理解了才往下走
> - 报错了先让 AI 分析原因，再让它修
>
> **预计耗时**：每天 2-3 小时，7-12 天完成全部 5 个阶段

---

## 一、给 AI 下指令的黄金模板

每次向 AI 提问时使用这个结构，输出质量和控制力都会大幅提升：

```
【背景】我现在做到第 X 步了，目前项目里有这些文件：[列出文件]

【目标】我想实现 XXX 这一个功能

【约束】请你：
  1. 一次只改 [N] 个文件，总共不超过 50 行代码
  2. 每段关键代码后面用 # 中文注释解释它在干什么
  3. 如果有多种实现方式，先列出选项让我选，不要直接写
  4. 写完后告诉我：在浏览器里怎么操作来验证功能正常

【当前代码】（粘贴相关代码片段，不超过 30 行）
```

### 实际示例

```
【背景】我已经做好了文件上传（main.py 的 /upload 路由），数据能正确显示前 10 行。
       现在在 main.py 和 upload.html 的基础上做下一步。

【目标】加一个缺失值检测功能：上传数据后跳转到清洗页面，显示每列有多少缺失值。

【约束】请你：
  1. 一次只改两个文件（路由 + 模板），控制在 50 行内
  2. 关键代码用中文注释解释
  3. 告诉我改完后在浏览器怎么操作来验证

【当前代码】
# main.py 的上传路由：
@app.post("/upload")
async def upload(file: UploadFile = File(...)):
    contents = await file.read()
    df = pd.read_csv(io.BytesIO(contents))
    ...
```

### 🟣 阶段 4：机器学习模块

> 🧠 **核心目标**：破除"机器学习=黑魔法"的误解。它就是找一条线贴近数据点——R² 越接近 1，线贴得越好。

#### 步骤 4.1：线性回归基础版

| 项目 | 内容 |
|------|------|
| **给 AI 的提示词** | *"用 scikit-learn 的 LinearRegression 帮我实现：用户在页面上选择特征列（可以多选）和目标列，数据按 80/20 划分训练集和测试集，训练模型后在页面上展示：回归系数表、R² 分数、MSE。解释什么叫训练集/测试集划分、R² 的含义是什么。"* |
| **验证方式** | 选择特征和目标列后，页面显示回归系数、R²（一个 0 到 1 之间的数）、MSE |
| **关键概念** | 训练集 vs 测试集；过拟合 vs 欠拟合；R² 解读（0.8 以上算不错） |

#### 步骤 4.2：预测值 vs 真实值可视化

| 项目 | 内容 |
|------|------|
| **给 AI 的提示词** | *"画一个散点图：X 轴是真实值，Y 轴是模型预测值。画一条 y=x 的对角线作为参考——完美预测的话所有点应该在对角线上。解释这张图怎么看，散点偏离对角线说明什么。"* |
| **验证方式** | 散点大致沿对角线分布；用很差的数据集训练时，散点明显偏离对角线 |
| **关键概念** | 残差（预测值 - 真实值）；对角线是"完美预测线" |

#### 步骤 4.3：用户输入新数据预测

| 项目 | 内容 |
|------|------|
| **给 AI 的提示词** | *"训练完模型后，加一个输入框让用户输入新样本的特征值（比如输入房屋面积和卧室数），点击'预测'按钮，显示预测结果。解释模型是怎么'记住'训练结果并用在新数据上的。"* |
| **验证方式** | 输入新数据 → 点击预测 → 返回一个合理的预测值（如预测房价，输入面积=150, 卧室=3，返回 ~300000） |
| **关键概念** | `model.predict()`；模型参数（系数和截距）的意义 |

#### 步骤 4.4：三种算法对比（扩展）

| 项目 | 内容 |
|------|------|
| **给 AI 的提示词** | *"加上 Ridge 和 Lasso 两种回归算法。做一个对比表，列出三种算法（线性回归 / Ridge / Lasso）的 R²、MSE、MAE。解释正则化 L1（Lasso）和 L2（Ridge）的区别，什么时候用哪个。"* |
| **验证方式** | 对比表出现三行数据，Ridge 和 Lasso 的 MSE 通常不等于线性回归（不一致才说明正则化在起作用） |
| **关键概念** | 正则化的目的——防止过拟合；L1 能做特征选择（某些系数变成 0） |

## 三、"看不懂"急救手册

| 你的困境 | 立刻这样问 AI |
|----------|-------------|
| 某段代码完全看不懂 | *"请逐行解释这段代码，用中文。当我是刚学 Python 一个月的人。"* |

## 六、终极原则重申

```
❌ "帮我写一个数据分析系统"     → 你什么都学不到
❌ "这段代码直接给我就行"        → 复制粘贴，下次还是不会
❌ "报错了，帮我修好"            → 你不知道为什么错

✅ "先讲思路，再写代码，加注释"   → 你学到了
✅ "这段我看不懂，拆开逐行解释"   → 你真正理解了
✅ "这个报错是什么意思？先分析"   → 你学会调试了
```

> **每完成一个阶段，试着给组员讲一遍这个模块做了什么。能讲清楚，才是真懂了。**
